# SWLS Final Learning Report v1.0.0

## 1. 学習方針
SWLSはモデルを自動改変する仕組みではない。実記事の出力、人間の修正、評価、原因分類をLearning Recordに保存し、複数記事で再発した事項のみを製品資産の改善候補へ昇格する。

## 2. UATで確認した学習
### 検索意図・タイトル
- 主検索意図を先頭で明示するタイトルは、読者に記事の回答範囲を伝えやすい。
- 情報を詰め込みすぎた長いタイトルは、主題を弱めるため整理が必要である。

### 導入・構成
- 導入で結論または到達点を示し、その後に条件・手順・比較軸を説明する流れが安定した。
- H2は検索者の判断順序に合わせると、冗長な重複を減らせる。

### FAQ
- FAQは本文の重複ではなく、関連する追加疑問、例外、失敗時の対処に限定する。
- FAQ追加は必須ではなく、検索意図を補完できる場合だけ行う。

### 内部リンク
- 同一語を含むだけの候補より、読者の次の行動に直接つながる記事を優先する。
- 関連性が弱い場合は候補数を満たすための提案をしない。

### 収益・既存資産保護
- 比較表、独自体験、PR表記、商品リンク、CTAは価値のある既存資産として保護する。
- 収益導線は読者の判断に必要な情報の後へ配置し、本文を広告中心にしない。

### Safety / YMYL
- 身体症状を霊的・超自然的な原因へ結びつける記事は、医療上の危険を招く可能性がある。
- 片側の脱力・しびれ等の緊急性が疑われる症状は、受診・救急相談を優先する。
- YMYLでは通常の記事生成よりSafety Gateを優先し、必要に応じてBLOCKEDまたはNEEDS_EXPERT_REVIEWとする。

## 3. 資産別反映
- Pattern Library：Intent-first導入、FAQ補完、内部リンク関連性、資産保護、YMYL安全パターンを共通ルールへ追加。
- Knowledge Base：実証済み編集原則とYMYLエスカレーション条件を追加。
- Runtime：Risk Assessmentを構成生成より前に固定し、Safety Gateの優先順位を明確化。
- Contract：v1.2を維持。現UATでは破壊的変更を必要とする事実は確認されなかった。
- Templates：Learning Record v1.1.0を正式採用。

## 4. 次期学習条件
v1.1候補へ昇格する変更は、原則として10件単位のLearning Report、人間確認、再発性、影響範囲、回帰テスト結果を必要とする。
